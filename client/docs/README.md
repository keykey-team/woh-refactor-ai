# 📐 Frontend Architecture Guidelines (FSD)

## 🎯 Цель

Установить единые правила архитектуры фронтенд-кода в команде,  
обеспечить предсказуемую структуру проекта, удобство поддержки, масштабируемость и безопасный рефакторинг.

---

# 1. Архитектурные принципы

## 1.1 Общие правила

- Зависимости разрешены только сверху вниз
- Запрещены циклические зависимости
- Любое исключение должно быть согласовано
- Путь к файлу должен объяснять его назначение
- Один файл — одна ответственность
- Дублирование логики запрещено
- Архитектура должна позволять безопасный рефакторинг
- Взаимодействие только через Public API
- Временный код запрещён
- `layout` как слой запрещён (только `app/` или `widgets`)
- `shared` и `app` не имеют слайсов
- Page-section → `pages`, переиспользуемые → `widgets`

---

## 1.2 Важные изменения

- ❌ `style` сегмент запрещён → всё в `ui`
- Page-only блоки → `pages/ui`
- Домен → `entities`, не `widgets`
- Максимум ~100 строк на файл
- ❌ папка `layout` запрещена

---

# 2. Layers (Слои)

## Общие правила

- Импорты только сверху вниз
- Каждый слой строго по назначению
- `shared` — ни от кого не зависит
- `entities` не знают про `features/widgets`
- `features` не знают про `widgets`
- `pages/app` — без бизнес-логики

---

## Слои

### app

Инициализация приложения

**Можно импортировать:**
- pages, widgets, features, entities, shared

**Запрещено:**
- бизнес-логика
- слайсы

---

### pages

Страницы и роутинг

**Можно:**
- widgets, features, entities, shared

**Запрещено:**
- бизнес-логика
- API вызовы

---

### widgets

Композиция UI

**Можно:**
- features, entities, shared

**Запрещено:**
- бизнес-логика
- API
- store

---

### features

Пользовательские действия

**Можно:**
- entities, shared

**Содержит:**
- бизнес-логику
- API
- сценарии

---

### entities

Доменные сущности

**Можно:**
- shared

**Содержит:**
- модели
- состояние
- UI одной сущности

---

### shared

Переиспользуемый код

**Содержит:**
- UI primitives
- utils
- config

**Запрещено:**
- бизнес-логика

---

# 3. Слайсы (Slices)

## Правила

- 1 слайс = 1 ответственность
- Есть один Public API (`index.ts`)
- Импорты только через Public API
- `shared` и `app` без слайсов

---

# 4. Сегменты (Segments)

### ui
- JSX, стили
- без логики

### model
- состояние
- бизнес-правила

### lib
- утилиты
- helper функции

### api
- HTTP запросы
- DTO

### config
- константы
- настройки

---

# 5. Public API

- вход только через `index.ts`
- нельзя обходить

---

# 6. Naming

- layers: app, pages, widgets, features, entities, shared
- slices: kebab-case
- segments: ui, model, lib, api, config

---

# 7. Ограничения

- ≤ 100 строк на файл (+20 максимум)

---

# ⚠️ Ключевой принцип

Архитектура = ограничения  
Нарушение = ошибка
